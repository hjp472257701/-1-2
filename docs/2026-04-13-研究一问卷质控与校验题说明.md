# 2026-04-13 研究一问卷质控与校验题说明

## 1. 调整目的

为降低被试猜测题目意图、随意作答或社会赞许偏差对结果的影响，研究一问卷采用“2 硬校验 + 2 软校验（含矛盾配对）”方案，并将判定结果写入导出数据，便于后续论文方法与数据清洗说明。

---

## 2. 校验题设置

## 2.1 硬校验（Instruction Check，2题）

- `ATTN01`：请认真阅读本题，并选择“4 符合我的特征”。
  - 期望答案：`4`
- `ATTN02`：请在本题选择“2 不符合我的特征”。
  - 期望答案：`2`

判定规则：两题全部正确才记为 `attention_passed = 1`。

## 2.2 软校验（Response Style Check）

- 社会赞许题：
  - `ATTN03`：我几乎从不对任何人产生负面情绪。
  - 触发条件：`ATTN03 >= 4` 记为 `soft_social_flag = 1`

- 矛盾配对题：
  - `ATTN04`：遇到不开心的事，我通常能较快调整情绪。
  - `ATTN05`：我经常会被小事激怒，且很难平静下来。
  - 触发条件：`ATTN04 >= 4` 且 `ATTN05 >= 4`，记为 `soft_conflict_flag = 1`

---

## 3. 题目呈现策略

- 校验题不会单独成组显示。
- 前端会将 `ATTN` 题随机插入其他题组中，降低可识别性。
- 题干避免出现“质量控制”“校验”等暴露目的的表述。

---

## 4. 质量分层规则（后端自动判定）

系统输出字段：

- `response_quality_level`：`valid` / `suspicious` / `exclude`
- `exclude_recommended`：`0/1`

判定逻辑：

1. `exclude`（建议剔除）
   - 任一硬校验失败（`attention_passed != 1`），或
   - 缺失题过多（`missing_count > 20%`），或
   - 作答时长过短（`duration_ms < 3分钟` 且 `duration_ms > 0`）
2. `suspicious`（可疑样本）
   - 硬校验通过，但触发任一软校验（`soft_social_flag=1` 或 `soft_conflict_flag=1`）
3. `valid`（有效样本）
   - 上述条件均未触发

其中 `exclude_recommended` 在 `exclude` 和 `suspicious` 时均记为 `1`，用于清洗时快速筛选。

---

## 5. 导出字段位置

- 研究一导出：`/api/export_research1.csv`
  - 含 `attention_correct`、`attention_total`、`attention_passed`
  - 含 `soft_social_flag`、`soft_conflict_flag`
  - 含 `response_quality_level`、`exclude_recommended`

- 合并导出：`/api/export_merged.csv`
  - 对应前缀字段：`q1_attention_*`、`q1_soft_*`
  - 同时给出 `response_quality_level`、`exclude_recommended`

## 6. 论文写作建议（可直接改写）

“本研究采用硬校验与软校验结合的问卷质控策略。硬校验由 2 道指令题构成，任何一题错误即判定为质量不通过。软校验用于识别社会赞许型与前后作答不一致型反应风格。基于作答完整性、作答时长及校验结果，系统将样本分为 valid、suspicious 与 exclude 三类，并输出 `exclude_recommended` 字段用于数据清洗与敏感性分析。”
